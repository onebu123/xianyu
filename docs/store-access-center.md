# 店铺接入中心使用说明

## 目标

第 3 轮把原来的演示接入链路升级为统一的店铺接入中心，面向以下三类实体：

- 店铺：平台店铺本身，支持闲鱼和淘宝。
- 店主账号：负责授权登录的账号主体，记录手机号、授权方式和最近授权时间。
- 系统操作员：后台登录用户，记录是谁发起授权、启停、健康检查和批量操作。

## 主要能力

- 支持多个闲鱼店铺和多个淘宝店铺并存。
- 支持店铺分组、标签、备注、启用、停用。
- 支持授权会话记录，区分待完成、已完成、已过期、已失效。
- 支持待激活、已激活、掉线、异常四类连接状态。
- 支持单店操作和批量操作。
- 支持店铺健康检查任务与日志回看。
- 支持对掉线或异常店铺重新授权。

## 页面结构

`/stores` 页面包含以下区域：

- 汇总卡片：展示总店铺数、平台分布、待激活、掉线、异常、停用等关键指标。
- 店铺列表：展示平台、店主账号、状态、分组标签、备注、同步时间和操作入口。
- 授权会话记录：查看新接入和重新授权的会话状态。
- 健康检查日志：查看手动或批量执行后的检查结果。

## 接入流程

### 新接入店铺

1. 在店铺接入中心点击“新增闲鱼店铺”或“新增淘宝店铺”。
2. 系统创建授权会话并拉起授权弹窗。
3. 在弹窗完成登录授权，系统写入店铺和店主账号记录。
4. 闲鱼店铺默认进入“待激活”；淘宝店铺默认进入“已激活”。
5. 对闲鱼店铺执行激活后，才会进入正式调度。

### 重新授权

1. 在掉线或异常店铺行点击“重新授权”。
2. 系统为当前店铺创建新的授权会话，并使旧的待处理会话失效。
3. 完成授权回调后，系统刷新原店铺授权状态，不新增重复店铺。

## 状态说明

### 连接状态

- `pending_activation`：授权成功，但还未手动激活。
- `active`：连接正常，可参与正式调度。
- `offline`：授权过期或同步失败，当前掉线。
- `abnormal`：接口异常或授权失效，需要人工复核。

### 授权状态

- `authorized`：授权有效。
- `pending`：授权会话等待完成。
- `expired`：授权会话或授权有效期已过。
- `invalidated`：原授权或原会话已失效。

### 健康状态

- `healthy`：检查通过。
- `warning`：待激活等预警状态。
- `offline`：检查确认掉线。
- `abnormal`：检查确认异常。
- `skipped`：因店铺停用跳过检查。

## 常用操作

### 编辑店铺信息

- 可修改分组、标签、备注。
- 标签支持中文顿号、逗号或空格分隔。

### 启用与停用

- 停用后店铺保留记录，但不再参与任务调度。
- 健康检查对停用店铺会返回 `skipped`。

### 批量操作

- 批量启用
- 批量停用
- 批量体检

## 故障排查

### 1. 授权弹窗没有打开

- 检查浏览器是否拦截弹窗。
- 确认当前用户角色为 `admin` 或 `operator`。
- 确认登录态没有过期。

### 2. 店铺显示掉线

- 先查看授权状态是否为 `expired`。
- 重新发起授权。
- 再执行一次健康检查确认状态是否恢复。

### 3. 店铺显示异常

- 查看最近一次健康检查说明。
- 检查是否存在授权失效或回调异常。
- 必要时重新授权并记录备注。

### 4. 店铺一直停留在待激活

- 说明授权已完成，但尚未执行激活。
- 在店铺列表点击“激活”后再观察最近同步时间。

### 5. 批量体检结果全部为 skipped

- 检查选中的店铺是否已被停用。
- 如需恢复调度，先批量启用，再重新执行体检。
