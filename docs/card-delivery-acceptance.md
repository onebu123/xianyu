# 第 5 轮验收记录

## 轮次

- 第 5 轮：卡密发货引擎
- 验收日期：2026-03-10

## 验收范围

- 卡密库存与批次模型
- 自动发货任务与失败重试
- 出库、补发、回收、撤回记录
- 低库存预警
- 模板联动
- 自动化测试

## 自动化验证

- `npm run lint`
- `npm run test`
- `npm run build`

## 核对结果

- 已新增 `card_batches`、`card_inventory_items`、`card_outbound_records`、`card_recycle_records`、`card_delivery_jobs`、`card_stock_alerts`。
- 已支持按卡种前缀、密码前缀和分隔符导入卡密，并区分新增、重复、格式异常。
- 已支持订单自动发货与幂等控制，重复发货会返回原出库记录。
- 已支持失败任务重试、手动补发、回收返库、撤回禁用。
- 已支持低库存预警和模板消息联动。
- 已把卡密工作台升级为库存模型、发货引擎和履约记录三个可操作视图。

## 自动化测试覆盖点

- 卡密导入、去重、格式校验、禁用与预警。
- 同一订单幂等发货与失败任务重试。
- 补发记录可追溯原订单和原出库单。
- 回收记录写入后可在记录页回查。

## 人工验收结论

- `card-delivery` 页面可直接执行导入样例、启停发货配置、重试失败任务。
- `card-records` 页面可以看到出库、补发、回收、批次导入三类记录。
- 出库成功后订单详情可以看到发货状态与时间线更新。

## 已知限制

- 当前模板联动采用统一启用模板，不按卡种拆分模板池。
- 当前演示版未覆盖复杂多卡组合订单，只覆盖单卡自动履约主线。
