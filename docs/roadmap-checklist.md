# 面向虚拟商品卖家的安全私有化闲鱼工作台严格交付计划表

## 使用规则

- 本文档是项目执行主清单，不是愿景文案。
- 每一轮只有在“必须完成项、交付物、验收标准、完成定义、完成证据”全部满足后，才允许把 `[ ]` 改成 `[x]`。
- 任何一项验收标准未通过，该轮都不得勾选完成。
- 任何一轮都不允许用“演示数据冒充生产能力”通过验收。
- 任何一轮都必须同步更新文档、接口说明、测试或验证记录。

## 全局打勾规则

### 打勾前必须同时满足

- 代码已落库，且本轮范围内的前后端功能都能运行。
- 本轮新增或修改的接口、页面、配置项都有文档。
- 本轮相关构建通过。
- 本轮相关测试通过；如果暂时没有自动化测试，必须补人工验收记录。
- 本轮的异常路径已经验证，不允许只验证成功路径。
- 本轮没有遗留“先手工处理、以后再补”的核心阻塞项。

### 一票否决项

- 仍然依赖写死的演示数据才能展示核心能力。
- 关键流程只有页面，没有真实后端状态流转。
- 关键数据无持久化或重启后丢失。
- 关键权限无控制，任何登录用户都能做高风险操作。
- 没有日志、没有审计、没有错误处理就宣称完成。

### 每轮必须留下的完成证据

- 代码变更
- 文档变更
- 至少一条构建或测试记录
- 至少一条人工验收记录
- 本轮完成结论和已知风险记录

### 交付物

- 权限模型说明
- 账号管理页面
- 审计日志页面
- 安全配置文档
