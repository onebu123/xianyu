# 项目概览

## 项目定位

`Sale Compass` 是一个基于开源组件二次开发的销售统计后台，用于复刻 `https://www.goofish.pro/sale/statistics` 的核心业务分析能力。由于目标页面需要登录，页面细节结合了公开登录页、操作手册与“销售统计后台”常见模式进行高可信还原。

当前交付状态为“首个私有化交付版”：

- 可运行
- 可部署
- 可验收
- 可交接

交付包默认不附带任何演示业务数据，也不包含对目标站私有接口的抓取与复用。

## 已实现模块

- 销售总览：GMV、订单数、客单价、转化率
- 今日统计：今日 GMV、订单、访客、新增客户
- 商品统计：在售商品、动销商品、低库存商品、分类数
- 订单统计：待发货、已发货、已完成、平均发货时长
- 售后统计：进行中售后、已完结售后、退款金额、售后完结率
- 销售趋势：GMV / 订单数 / 退款金额趋势
- 来源分布：自然成交、客服跟进、复购推荐、短视频导流
- 订单明细：筛选、分页、CSV 导出
- 商品分析：分类销售额与商品排行
- 客户分析：地区分布与高价值客户
- 安全权限：管理员 / 运营 / 客服 / 财务四类角色、菜单隔离、按钮隔离、接口鉴权、审计日志、敏感配置脱敏

## 初始化管理员

- 首次启动通过 `.env` 中的 `APP_INIT_ADMIN_*` 创建初始化管理员
- 默认不再内置演示账号

## 技术栈

- 前端：React 19、Vite 7、Ant Design 5、Ant Design Pro Components、Apache ECharts
- 后端：Fastify 5、better-sqlite3、zod
- 数据：SQLite 本地库，首次启动按运行模式初始化

## 数据边界

- 默认使用本地 SQLite 数据库
- `prod` 模式首次启动只初始化管理员，不写入演示业务数据
- 只有显式切到 `demo` 模式时才会写入演示数据
- 普通重启不会重复重建数据库

## 部署边界

- 标准部署入口设计为 `http://<VPS_IP>:4300`
- 当前阿里云演示机由于公网 `4300` 链路未到达 ECS 网卡，实际入口采用 `http://47.251.63.59/sale-compass/`
- 该兼容方案通过 Nginx 路径代理实现，不影响现有根路径业务

## 关键目录

- `web/`：前端源码
- `server/`：后端源码与数据库初始化逻辑
- `deploy/`：现网代理配置样例
- `docs/`：交付文档
