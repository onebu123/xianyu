# 卡密发货引擎说明

## 目标

第 5 轮把卡密仓库从“库存展示”升级为“真实履约引擎”。当前实现已经覆盖：

- 卡种、批次、库存、出库、回收、任务、预警模型
- 卡密导入、去重、格式校验、禁用/恢复
- 订单锁卡与幂等发货
- 自动发货、手动补发、回收返库、撤回禁用
- 失败重试和低库存预警
- 模板内容与发货消息联动

## 数据模型

核心表如下：

- `card_batches`：导入批次，记录新增、重复、格式异常统计
- `card_inventory_items`：单张卡密库存，状态包含 `available`、`locked`、`sold`、`disabled`
- `card_outbound_records`：出库记录，关联订单、库存项、模板、补发链路
- `card_recycle_records`：回收与撤回记录
- `card_delivery_jobs`：自动发货任务与失败重试入口
- `card_stock_alerts`：低库存预警记录

## 关键规则

### 导入规则

- 按卡种配置的 `card_prefix`、`password_prefix`、`separator_text` 解析卡密
- 同一卡种下 `card_no + card_secret` 唯一
- 导入结果会区分新增、重复、格式异常三类

### 发货规则

- 只有已绑定卡密商品的订单可进入自动发货
- 当前演示版本按单卡订单处理
- 若同一订单已存在有效出库记录，再次发货会直接返回原记录，保持幂等
- 发货成功后同步写入：
  - `card_outbound_records`
  - `orders` 发货状态与主状态
  - `order_events` 时间线

### 补发与回收

- 补发会新建一条 `resent` 出库记录，并通过 `parent_outbound_id` 回溯原记录
- 回收会把库存返还为 `available`
- 撤回会把库存改为 `disabled`
- 回收和撤回都会写入 `card_recycle_records`

### 预警

- 当前低库存阈值为 `40`
- 库存变化后会同步刷新 `card_stock_alerts`

## 工作台入口

### `card-types`

- 查看卡种库存模型
- 查看最近导入、最近出库、可用/锁定/已售/禁用数量

### `card-delivery`

- 查看发货配置
- 导入样例卡密
- 禁用或恢复样卡
- 执行待处理任务或重试失败任务
- 查看低库存提醒

### `card-records`

- 查看出库记录
- 查看补发链路
- 查看回收记录
- 查看批次导入统计

## 已知限制

- 当前演示版默认按单卡订单处理，多张卡组合订单会放到后续轮次扩展
- 当前模板联动采用统一启用模板，不区分卡种专属模板池
