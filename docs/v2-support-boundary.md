# v2.0.0 支持范围与边界

## 支持范围

- 支持私有化单租户部署和 SaaS 多租户控制面部署。
- 支持 PostgreSQL 作为企业正式数据库基线。
- 支持 SQLite 作为本地开发、演示和历史私有化兼容模式。
- 支持 Redis 队列、独立 Worker、任务重试、幂等键和死信状态。
- 支持平台账号、租户成员关系、管理员 MFA、密码策略和审计日志。
- 支持多店铺运营、订单、售后、资金、AI 客服、AI 议价和开放平台基础能力。

## 不承诺范围

- 不承诺绕过闲鱼官方风控或协议限制。
- 不承诺自动完成所有企业自助注册、计费、渠道分账和合同流程。
- 不承诺第一阶段接入 SSO、OIDC、LDAP 或 SAML。
- 不承诺 SQLite 在线无停机迁移到 SaaS 多租户。
- 不承诺共享库多租户方案。
- 不承诺在未提供真实账号、真实店铺授权和合法接入条件时完成真实闲鱼生产联调。

## 生产接入前置条件

- 企业客户必须确认账号授权、数据合规、操作边界和风控责任。
- 必须配置生产 PostgreSQL、Redis、备份目录、日志目录和监控采集。
- 必须使用强密码、管理员 MFA、密钥加密和最小权限账号。
- 必须先完成沙箱或小流量试运行，再扩大到正式店铺。

## 响应级别

- `P0`：登录、租户隔离、数据丢失、支付资金、系统不可用。
- `P1`：核心业务不可完成但存在人工绕行。
- `P2`：单模块异常、报表偏差、任务延迟。
- `P3`：体验优化、文案、非核心配置改进。
