# 企业级改造路线图

当前主线已收口到 `v2.0.0`，目标是对标企业级闲鱼商家运营平台，提供 SaaS + 私有化双模式交付能力。

## 已完成

- [x] 控制面与租户模型。
- [x] 平台账号、租户成员关系和两段式登录。
- [x] 每租户独立业务库治理。
- [x] PostgreSQL 企业正式基线。
- [x] Redis 队列与独立 Worker。
- [x] 多店铺运营中心。
- [x] 订单、售后、资金、AI 客服、AI 议价核心边界。
- [x] 开放平台、API Key、Webhook 和审计。
- [x] MFA、密码策略、密钥引用和敏感配置隔离。
- [x] OpenAPI、前端冒烟、CI 矩阵和发布打包。

## v2 交付架构

- 控制面：租户、平台账号、成员关系、开通任务、密钥引用。
- 租户面：业务接口、租户数据库、店铺、订单、售后、资金、AI 和开放平台。
- 任务面：Redis 队列、Worker、重试、幂等、死信和任务审计。
- 运维面：健康检查、指标、日志、备份恢复、回滚和事故响应。

## 后续版本方向

- `v2.1`：真实生产店铺小流量联调和风控频率治理。
- `v2.2`：企业自助开通、计费、套餐和用量统计。
- `v2.3`：SSO、OIDC、LDAP、SAML 企业身份集成。
- `v2.4`：更多外部 ERP、财务系统和客服系统连接器。

## 验收标准

- `npm run release:v2` 通过。
- 私有化和 SaaS 两种模式均可启动、登录、验权和冒烟。
- 新租户可创建独立业务库。
- 队列任务可恢复、可重试、可审计。
- 文档、版本号、发布包和 Git 标签统一为 `v2.0.0`。
