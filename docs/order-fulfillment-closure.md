# 订单与发货闭环说明

## 目标

第 7 轮把订单中心从“能看订单”升级为“能统一履约和处理异常”。当前实现已经覆盖：

- 统一履约类型识别，打通普通订单、卡密订单、直充订单
- 统一履约队列，收敛待处理、处理中、成功、失败、待人工五类状态
- 异常订单工作台
- 履约任务日志
- 人工重试、补发、终止、备注
- 店铺维度履约表现统计

## 统一履约模型

订单列表和订单详情新增以下统一履约字段：

- `fulfillmentType`：`standard`、`card`、`direct_charge`
- `fulfillmentQueue`：`pending`、`processing`、`success`、`failed`、`manual_review`
- `fulfillmentStage`：当前履约阶段标题
- `fulfillmentStageDetail`：当前履约阶段说明

订单详情额外提供：

- `fulfillment`：当前履约摘要，包含任务号、供应商单号、出库单号、重试次数、人工原因和动作能力
- `fulfillmentLogs`：从 `order_events` 中抽出的履约相关日志

## 工作台组成

### 异常订单工作台

- 统一聚合 `failed` 和 `manual_review` 队列
- 直接展示订单号、店铺、商品、履约类型、当前阶段和更新时间
- 支持从工作台直接进入详情、重试和卡密补发

### 履约任务日志

- 统一读取 `order_events` 中的 `card_*`、`direct_charge_*`、`fulfillment_*` 事件
- 订单中心详情抽屉与工作台共用同一套履约日志源
- 人工终止、备注、补发等动作会同步留下日志和审计记录

### 店铺履约表现

- 以店铺为维度汇总总订单数、成功数、失败数、人工数、处理中数
- 同时返回成功率、失败率、人工率
- 统计维度与当前订单筛选条件保持一致

## 人工动作

### 重试

- 卡密订单会重跑自动发货或失败任务
- 直充订单会重新下发或进入重试链路

### 补发

- 当前仅卡密履约支持统一补发入口
- 补发会继续复用原出库链路并写入履约日志

### 终止

- 终止后订单主状态更新为 `closed`
- 发货状态统一切到 `manual_review`
- 对应卡密 / 直充任务会记录失败原因
- 统一写入 `fulfillment_terminated` 事件

### 备注

- 备注会同步写入 `orders.seller_remark`
- 同时追加 `fulfillment_note` 事件，便于后续审计和排障

## 前端入口

订单中心页面新增：

- 履约队列摘要卡片
- 异常订单工作台
- 店铺履约表现表格
- 履约任务日志表格
- 订单详情抽屉中的履约控制区和履约日志时间线

## 已知限制

- 当前统一补发仅支持卡密订单，直充订单仍以重试和人工接管为主
- 当前普通实物订单只进入统一履约队列，不包含单独的自动任务引擎
