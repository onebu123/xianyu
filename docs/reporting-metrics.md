# 第9轮 统计口径说明

## 目标

第 9 轮的目标是把统计与报表从演示口径切换到真实业务口径，确保页面、接口和导出使用同一套指标定义，并且每个指标都可以回溯到原始记录。

## 数据来源

- 订单主表：`orders`
- 支付记录：`order_payments`
- 商品成本：`products.cost`
- 统一履约状态：订单履约元数据、卡密任务、直充任务
- 售后主单：`after_sale_cases`
- 退款单：`after_sale_refunds`
- 争议补偿：`after_sale_disputes`

## 核心指标

### 订单原额

- 公式：`SUM(order_payments.gross_amount)`
- 说明：表示优惠前的支付原额。

### 优惠金额

- 公式：`SUM(order_payments.discount_amount)`
- 说明：表示营销让利金额。

### 实收金额

- 公式：`SUM(order_payments.paid_amount)`
- 说明：当前报表的基础收入口径。

### 退款金额

- 公式：`SUM(orders.refund_amount)`
- 说明：已回写到订单主线的退款金额。

### 净销售额

- 公式：`实收金额 - 退款金额`
- 说明：用于店铺、商品、订单维度的统一净收入。

### 商品成本

- 公式：`SUM(products.cost * orders.quantity)`
- 说明：按订单销量回溯商品成本。

### 毛利

- 公式：`净销售额 - 商品成本`
- 说明：反映商品经营毛利，不扣售后补偿。

### 毛利率

- 公式：`毛利 / 净销售额 * 100%`
- 说明：净销售额为 0 时返回 0。

### 售后补偿

- 公式：`SUM(after_sale_disputes.compensation_amount)`
- 说明：用于计算净利润的售后额外损耗。

### 净利润

- 公式：`毛利 - 售后补偿`
- 说明：作为经营复盘的最终利润口径。

## 维度统计

### 店铺维度

- 订单数
- 实收金额
- 退款金额
- 净销售额
- 毛利 / 毛利率
- 售后单数
- 履约成功率
- 人工处理单
- 平均发货时长

### 商品维度

- 订单数
- 销量
- 实收金额
- 退款金额
- 净销售额
- 毛利 / 毛利率
- 售后单数
- 履约成功率

### 订单维度

- 订单总数
- 已完成订单
- 售后订单
- 退款订单
- 履约成功率
- 人工处理单
- 平均发货时长
- 订单主状态分布
- 订单来源分布
- 履约队列分布

### 售后维度

- 售后总单量
- 处理中售后
- 已完结售后
- 已驳回售后
- 超时售后
- 退款金额
- 售后完结率
- 售后类型分布
- 售后状态分布

## 筛选规则

- 支持 `today`、`last7Days`、`last30Days`、`last90Days`
- 支持自定义 `startDate` / `endDate`
- 支持 `storeIds` 多店铺筛选，使用逗号分隔
- 支持单商品、分类、来源、关键词过滤

## 一致性约束

- 页面展示数据来自 `/api/reports`
- CSV 导出来自 `/api/reports/export`
- 导出方法内部直接复用 `/api/reports` 的同一套聚合结果
- 首页 `/api/dashboard` 的摘要卡已同步切换到利润口径
