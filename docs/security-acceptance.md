## 人工验收记录

### 角色隔离

- 登录页不再展示演示账号，首次登录使用初始化管理员；其余角色由管理员在系统内创建。
- 管理员登录后可访问 `system-accounts`、`open-logs`、`system-configs` 三个安全模块。
- 运营登录后可访问店铺、商品、订单、卡密、分销等模块，但不能访问账号管理、审计日志、敏感配置。
- 客服登录后仅保留统计、订单、售后、客户相关模块，不显示店铺管理、资金审核和系统安全配置入口。
- 财务登录后可访问报表和资金中心，并可执行提现审核，不显示店铺管理与系统安全配置入口。

### 权限拒绝与审计留痕

- 用运营账号访问 `system-accounts` 接口时返回 `403`。
- 用客服账号执行提现审核接口时返回 `403`。
- 登录失败、越权访问、账号停用后再次登录、敏感配置更新均会进入审计日志。
- 审计日志可查看操作者、时间、目标对象、结果、来源 IP 与脱敏说明。

### 敏感配置保护

- 安全配置页面表格仅展示 `maskedValue`，不回显明文。
- 更新密钥后数据库仅保存 `value_encrypted` 与 `value_masked`。
- 审计日志中不记录明文密钥，只记录脱敏值与动作结果。

## 结论

- 第2轮“安全与权限体系”所要求的登录安全、权限隔离、配置保护、审计可追踪已经具备。
- 当前未包含 SSO 与企业级自定义 RBAC，符合本轮边界定义。
